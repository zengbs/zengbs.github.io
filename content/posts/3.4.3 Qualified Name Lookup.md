---
title: 3.4.3 Qualified Name Lookup
tags: [CPP]

---

1. A name prefixed by the unary scope operator :: is looked up in global scope, in the translation unit where it is used. The name shall be declared in global namespace scope or shall be a name whose declaration is visible in global scope because of a using-directive. The use of :: allows a global name to be referred to even if its identifier has been hidden.
2. If a pseudo-destructor-name contains a nested-name-specifier, the type-names are looked up as types in the scope designated by the nested-name-specifier.
    * In a qualified-id of the form: 
     `:: nested-name-specifier ~ class-name`
where the nested-name-specifier designates a namespace scope, and
    * In a qualified-id of the form:
    `:: nested-name-specifier class-name :: ~ class-name`
the class-names are looked up as types in the scope designated by the nested-name-specifier.

	```c++
	#include <iostream>
	
	class C {
	    public:
	    typedef int I; // C::I is a type alias for int
	};
	
	// Global type aliases
	typedef int I1, I2;
	
	int main() {
	    int* p = new int; // Pointer to int
	    int* q = new int; // Another pointer to int
	
	
	    // Case1: nested-name-specifier ~ class-name
	    // nested-name-specifier → q->I1:: (global)
	    // ~ class-name → ~I2
	    q->I1::~I2();   // I2 is looked up in the scope of the global typedef (q->I1)
	
	
	    // Case2: nested-name-specifier class-name :: ~ class-name
	    // nested-name-specifier → p->C::
	    // class-name → I
	    // ~ class-name → ~I
	    p->C::I::~I();  // I is looked up in the scope of C (p->C::I)
	
	    std::cout << "Pseudo-destructor calls executed successfully.\n";
	
	    delete p;
	    delete q;
	
	    return 0;
	}
	```
